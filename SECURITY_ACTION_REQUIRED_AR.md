# إجراء أمني مطلوب قبل رفع المشروع

عُثر في الحزمة الأصلية على مفتاح خاص لتوقيع بيانات JSON، وAndroid JKS، وملف يحتوي قيم GitHub Secrets. حُذفت هذه الملفات من النسخة المنقحة، لكن الحذف لا يجعل القيم القديمة آمنة.

## المطلوب

1. أنشئ زوج مفاتيح جديدًا لتوقيع البيانات، واستبدل المفتاح العام المضمّن في التطبيق، ثم أعد توقيع بيانات `verified-data` بالمفتاح الجديد.
2. غيّر كلمات مرور Android Keystore. إذا لم يُنشر التطبيق بعد، أنشئ Keystore جديدًا. إذا كان منشورًا، لا تستبدل مفتاح التوقيع عشوائيًا؛ اتبع آلية إدارة/تغيير مفتاح الرفع المعتمدة لحساب النشر.
3. استبدل جميع GitHub Actions Secrets الخمسة بقيم جديدة.
4. إذا رُفعت الملفات إلى GitHub في أي Commit، راجع سجل Git ونظّفه بأداة مناسبة، ثم نسّق إعادة الاستنساخ مع كل من لديه نسخة من المستودع.
5. فعّل Secret Scanning وPush Protection إن كانا متاحين، ولا تحفظ نسخة مفاتيح غير مشفرة بجانب المشروع.

لا تستخدم المفاتيح الموجودة في الحزمة الأصلية للإنتاج بعد الآن.
