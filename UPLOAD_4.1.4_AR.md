# رفع الإصدار 4.1.4 — إصلاح القراءات اليومية

هذا الإصدار يعالج السبب الحقيقي لبطاقات البروكيمنن والرسالة والإنجيل الفارغة.

## ما تغيّر

1. يحوّل مرجع القراءة الذي اكتشفه التقويم إلى مرجع كتابي قياسي قبل أن تمسحه بوابة اللغة.
2. يحمّل نص الرسالة والإنجيل حرفيًا من مصدر مستقل لكل لغة:
   - العربية: Arabic Van Dyck.
   - الإنجليزية: World English Bible Classic.
   - اليونانية: Byzantine Greek New Testament 1904.
3. يعيد تركيب خدمة القداس بعد تعبئة النصوص، حتى يظهر النص داخل شاشة «القراءات» وداخل «القداس الإلهي» معًا.
4. يوقف النشر إذا كانت الرسالة أو الإنجيل ناقصة في أي لغة، أو إذا لم يظهر النص داخل تركيب القداس.
5. لا يخمّن البروكيمنن: عند عدم توفر نص يومي موثّق تظهر رسالة واضحة بدل بطاقة بيضاء.

## طريقة الرفع

فك ضغط حزمة ملفات الاستبدال، ثم انسخ محتوياتها إلى **جذر مستودع المشروع** مع الموافقة على استبدال الملفات. لا ترفع ملف ZIP نفسه.

بعد النسخ افتح `app/build.gradle.kts` وتأكد من وجود:

```kotlin
versionCode = 41004
versionName = "4.1.4"
```

ثم نفّذ:

```bash
git add .
git commit -m "fix(readings): publish complete native Scripture and rebuild service overlays"
git push
```

## الترتيب الصحيح بعد الـPush

1. افتح **Actions → Update → Run workflow**.
2. اختر `mode: update` واترك التاريخ فارغًا.
3. يجب أن ينجح فحص اكتمال الرسالة والإنجيل في العربية والإنجليزية واليونانية، وفحص ظهور النص داخل خدمة القداس.
4. بعد نجاح Update شغّل Workflow البناء، أو استخدم البناء الذي بدأ بعد الـPush.
5. ثبّت APK وتأكد من أن الإعدادات تعرض الإصدار `4.1.4`.
6. افتح **القراءات** ثم **القداس الإلهي** وتحقق من ظهور الرسالة والإنجيل في المكانين.

## ملاحظة البروكيمنن

الإصدار يصلح البطاقة البيضاء. إذا لم يتوفر بروكيمنن يومي مطابق من مصدر ليتورجي موثّق، سيظهر تنبيه صريح بدل اختراع نص أو استخدام نص يوم آخر.
