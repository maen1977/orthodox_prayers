# R17 — موثوقية التحديث اليومي الموقّع

يرفع هذا التحديث الإصدار إلى **5.0.13** ورقم البناء إلى **50013**.

## أهم التعديلات

- جدولة تحديث التطبيق عند **00:05 بتوقيت عمّان** بواسطة WorkManager دون طلب إذن المنبهات الدقيقة.
- إبقاء المهمة منتظرة تلقائيًا عند غياب الإنترنت، مع إعادة المحاولة وفق سياسة تصاعدية.
- منع فحص الشبكة عند كل عودة للتطبيق؛ فحص تصحيحات اليوم نفسه لا يتكرر قبل مرور 30 دقيقة.
- إضافة `data/update-manifest.json` موقّع بالمفتاح نفسه، ويحتوي التاريخ ورقم المراجعة والحد الأدنى لإصدار التطبيق والحجم وبصمة SHA-256 لكل مسار لغة.
- رفض أي مراجعة أقدم من مراجعة سبق أن قبلها التطبيق للتاريخ نفسه، وعدم الرجوع إلى مسارات قديمة بعد قبول Manifest موثوق لليوم.
- تثبيت الحد الأدنى المتوافق لقارئ البيانات داخل `canonical/update_contract.json` حتى لا يرتفع تلقائيًا مع كل إصدار تطبيق.
- جعل رقم المراجعة متزايدًا حتى عند إعادة تشغيل GitHub Actions في اليوم نفسه.
- فحص تطابق `today.json` والنسخة المؤرخة ونسخة أصول Android وكل مسار لغة وتوقيعاتها قبل النشر وبعده ومن النسخة المستنسخة من GitHub.

## الفحص

```bash
python scripts/verify_r17_patch.py
python -m unittest discover -s tests -p "test_*.py"
```

## نتيجة الفحص في هذه الحزمة

- نجح **135 اختبار Python**.
- نجحت فحوص الأسرار والتوقيعات والمخطط واللغات والمصادر وبيانات Android المضمنة.
- نجحت محاكاة بناء Manifest وتوقيعه والتحقق منه بمفتاح اختبار مؤقت.
- لم يُشغّل اختبار Gradle الكامل محليًا لأن Gradle 8.13 غير مخزن في البيئة وتعذر الوصول إلى `services.gradle.org`.
