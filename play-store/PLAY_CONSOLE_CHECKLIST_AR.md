# قائمة تجهيز Google Play — Orthodox Prayers v3.5.0

## جاهز داخل المشروع

- معرّف الحزمة `com.orthodoxprayers.privateapp`.
- `compileSdk` و`targetSdk` على API 35، والحد الأدنى API 26.
- بناء APK/AAB عبر GitHub Actions دون تخزين مفتاح Android في المستودع.
- R8 وتصغير الموارد في Release، ومنع HTTP غير المشفر.
- توقيع مستقل لبيانات اليوم والاحتفاظ بآخر نسخة موثوقة.
- ثلاث قنوات محتوى ديني مستقلة دون ترجمة أو fallback بين اللغات.

## شرط الإصدار الإنتاجي

الإصدار الموقّع محجوب حاليًا عمدًا إلى أن تنجح `scripts/validate_release_readiness.py`. يلزم قبل Production:

- اكتمال حزم الخدمات الرسمية للغات الثلاث.
- استيراد مكتبة كتاب مقدس رسمية غير فارغة لكل لغة.
- توفر نص الرسالة والإنجيل الحاليين حرفيًا في اللغات الثلاث.
- نجاح اختبارات Android وLint والمحاكي.

## أسرار GitHub Environment `production`

- `ANDROID_KEYSTORE_B64`
- `ANDROID_KEYSTORE_PASSWORD`
- `ANDROID_KEY_ALIAS`
- `ANDROID_KEY_PASSWORD`

بعد اكتمال بوابة المحتوى، أنشئ Tag مطابقًا لرقم النسخة مثل `v3.5.0`. الناتج المتوقع للمتجر هو AAB يحمل رقم النسخة المشتق من Gradle.

## خطوات Play Console

1. أنشئ التطبيق وثبّت معرّف الحزمة قبل أول رفع.
2. فعّل Play App Signing واحفظ مفتاح الرفع ونسخته الاحتياطية بأمان.
3. أدخل بيانات المتجر وسياسة الخصوصية العامة.
4. راجع Data safety من AAB النهائي.
5. ارفع أولًا إلى Internal testing.
6. اختبر اللغات الثلاث، البحث، الصيام، القراءات، الخدمات، الوضع دون إنترنت، والتحديث اليومي.
7. عالج أي crash أو ANR في Pre-launch report.
8. ابدأ بإطلاق تدريجي بعد اكتمال المراجعة الدينية والتقنية.

## عناصر يوفّرها المالك

حساب مطور صالح، بريد دعم، رابط خصوصية عام، لقطات شاشة حقيقية، بيانات المتجر، وإقرار الحقوق والموافقات النهائية.
