# إصلاح تجميع Java — Orthodox Prayers 5.0.1 R3

## سبب الفشل

تحولت محارف السطر الجديد `\n` في ثلاثة مواضع إلى أسطر فعلية داخل النص البرمجي، فنتجت أخطاء `unclosed string literal` في:

- `ReaderAdapter.java`
- `ReaderScreen.java`

## الإصلاح

- إعادة كتابة محارف السطر الجديد بصيغة Java الصحيحة `"\n"`.
- إبقاء الشرطة الطويلة `—` داخل النص البرمجي، لا خارجه.
- إحاطة إضافة المصدر الموثق بكتلة واضحة.
- إضافة اختبار آلي يفحص جميع ملفات Java ويمنع انتهاء أي سطر داخل String أو Char غير مغلق.
- إضافة اختبارات مباشرة لنص مشاركة المقطع وتذييله.

## التحقق
