# Orthodox Prayers 5.0.16 R20 — صلاة يومية موثوقة بثلاث لغات

تطبيق أندرويد أرثوذكسي يساعد المؤمن في متابعة قراءات اليوم، الصيام، المناسبات، الصلوات والخدمات، والبحث في النصوص المتاحة دون اتصال بعد تنزيل التحديث الموقّع.

## المبدأ الديني غير القابل للتجاوز

المشروع يحتوي ثلاثة مسارات مستقلة تمامًا:

- **العربية:** نص عربي رسمي فقط، بالأولوية: مطرانية الروم الأرثوذكس في الأردن، ثم بطريركية القدس العربية، ثم بطريركية أنطاكية العربية.
- **الإنجليزية:** نص إنجليزي منشور أصلًا من جهة أرثوذكسية رسمية مسجلة.
- **اليونانية:** نص يوناني منشور أصلًا من جهة كنسية يونانية أو أرثوذكسية رسمية مسجلة.

لا يترجم التطبيق نصًا دينيًا من لغة إلى أخرى، ولا يستخدم الذكاء الاصطناعي لإعادة الصياغة، ولا يضيف التشكيل أو النبرات آليًا. إذا لم يوجد نص رسمي أصلي للغة المختارة، يظهر **غير متوفر** بدل نص بديل.

## حماية النصوص

- يبقى نص العرض كما أُدخل من نسخة المصدر، مع المحافظة على محارف Unicode والتشكيل والنبرات.
- لكل نص وبند بحث بصمة SHA-256.
- تبسيط التشكيل أو الهمزات يحدث داخل فهرس بحث مشتق فقط، ولا يُعرض للمستخدم.
- بيانات اليوم لا يقبلها التطبيق إلا بعد التحقق من توقيع RSA/SHA-256.
- عند فشل التحديث تبقى آخر نسخة موقعة وسليمة.

## GitHub Actions

يوجد Workflowان فقط:

1. **Update** — تحديث يومي واحد عند الساعة 00:00 منتصف الليل بتوقيت `Asia/Amman`، مع تحقق النشر ضمن نفس التشغيل.
2. **Build** — بوابة الجودة، ثم استيراد أحدث بيانات منشورة موقعة والتحقق منها، واختبارات Android، وLint Debug، وبناء Debug APK. يبقى بناء APK/AAB الموقّع محميًا للإصدارات فقط، بينما CodeQL والمحاكي متوقفان مؤقتًا حتى يستقر البناء الأساسي.

عند تعذر مصدر يومي جديد يفشل التحديث بصورة مغلقة وتبقى آخر نسخة موقعة سليمة. ويعيد `scripts/validate_release_readiness.py` فحص اكتمال حزم الخدمات ونصوص الرسالة والإنجيل ومصادرها قبل كل إصدار Production.


## ميزات 5.0.16 R20

> R20 يفصل بين سلامة الحقول التقنية والاكتمال الكنسي الحقيقي، ويمنع إصدار Production ما دامت خدمة إلزامية ناقصة أو مختصرة أو غير مثبتة على طبعة أصلية كاملة.

- إزالة ترميز USFM العادي والممتد من العرض مع إبقاء كلمات المصدر كما هي.
- إدراج البروكيمنن والرسالة والإنجيل عبر مواضع ثابتة تعمل في كتب العربية والإنجليزية واليونانية.
- كل ملف يومي موقّع يحمل نصوص لغته ودليل مصدرها فقط.
- لا يرجع دليل الكنائس أو الروابط إلى اسم بلغة أخرى.
- قائمة اكتمال كنسي من 15 خدمة إلزامية، مستقلة عن عدّ الحقول.

### تجربة الاستخدام R19

- تعرض شاشة اللغات نسبة كل حزمة من ملفها الأصلي المستقل، لا من اللغة المفتوحة حاليًا.
- تظهر العربية والإنجليزية واليونانية معًا في ملخص اكتمال واضح.
- تستخدم التواريخ والأوقات لغة التطبيق المختارة بدل لغة الهاتف المختلفة.
- تُخفى رموز التشخيص والبصمات والمراجع التقنية افتراضيًا خلف زر اختياري.
- اختيار وقت التذكير وساعات الهدوء يتم من منتقي وقت حقيقي.
- يمكن إعادة إعدادات القراءة وحدها للوضع الافتراضي دون حذف المفضلة أو الملاحظات أو السجل.
- بقيت حماية R18.4 لمراجع Digital Chant Stand وقفل سلطة الأردن كما هي.

### مركز المصادر R18

- موصلات مستقلة للمصادر الرسمية الأردنية والمقدسية والأنطاكية واليونانية والإنجليزية.
- سلّم سلطة يجعل مطرانية الأردن المرجع المحلي وبطريركية القدس مرجع الاختصاص والتقويم.
- سجل صحة يومي لكل موصل، واكتشاف التعارض بين مراجع الرسالة والإنجيل.
- دليل كنائس أردني وروابط بث وتقويم وخدمات رسمية مؤرخة.
- بحث موحّد في الصلوات والقراءات والمصادر والكنائس مع تطبيع العربية وتحمل خطأ إملائي بسيط.
- تدقيق صريح لاكتمال القداس؛ لا يُعلن النص كاملاً ما دامت القطع المتغيرة الموثقة ناقصة.
- لا توليد ولا ترجمة آلية للنصوص الليتورجية، ولا نسخ للنصوص المقيدة دون إذن.


- تقويم كنسي شهري مع فتح التفاصيل الموثوقة المتاحة فقط، واختيار عرض غريغوري أو يولياني قديم.
- تذكيرات اختيارية لصلاة الصباح والمساء وقراءات اليوم والأعياد والصيام وتذكير شخصي.
- سجل لآخر 20 نصًا، وتثبيت النصوص في الصفحة الرئيسية، ومجموعات للمفضلة مع إعادة الترتيب.
- إعدادات قراءة موسعة: نوع الخط، تباعد الأسطر، التمرير التلقائي، حجم الخط، الوضع الليلي، ومنع انطفاء الشاشة.
- إدارة اللغات النشطة دون ادعاء حذف الملفات المضمّنة من APK.
- تشغيل تحديث التطبيق عند 00:05 بتوقيت عمّان بواسطة WorkManager مع انتظار الشبكة وإعادة المحاولة، دون إذن المنبهات الدقيقة.
- استخدام بيان تحديث موقّع يتضمن تاريخ البيانات ورقم المراجعة والحجم وبصمة SHA-256 لكل لغة، مع منع الرجوع إلى مراجعة أقدم.
- فحص تصحيحات اليوم نفسه كل 30 دقيقة كحد أقصى بدل الاتصال عند كل رجوع للتطبيق.
- أدوات آمنة لدمج دفعات نصوص إنجليزية أو يونانية أصلية ومصرّح بها، وتقارير دقيقة بالحقول الناقصة.

## البدء

اقرأ بالترتيب:

1. `GITHUB_UPLOAD_AR.md`
2. `GITHUB_SECRETS_AR.md`
3. `SETUP_AR.md`
4. `NATIVE_CONTENT_IMPORT_AR.md`
5. `RELEASE_READINESS_AR.md`

فحص المشروع محليًا:

```bash
python -m pip install -r requirements-dev.txt
python scripts/run_quality_gate.py --strict-native-lanes
```

الخيار `--strict-native-lanes` يرفض نسخ العربية إلى مسار آخر ويرفض النص المكتوب بأبجدية خاطئة. وتتحقق بوابة الإصدار مستقلًا من اكتمال النصوص الأصلية، بينما يمنع `validate_daily_ui_localizations.py` أي نقص في عناوين وتواريخ وصوم ومراجع واجهة العربية والإنجليزية واليونانية.

التحقق من Gradle Wrapper:

```bash
python scripts/verify_gradle_wrapper.py
```

إنشاء ZIP نظيف قابل للرفع، دون `.git` أو الكاش أو مفاتيح خاصة:

```bash
python scripts/create_clean_source_archive.py ../OrthodoxPrayers-5.0.16-R20-source.zip
```

ولإنشاء مشروع كامل يُفك مباشرة فوق جذر مستودع موجود دون إنشاء مجلد متداخل:

```bash
python scripts/create_clean_source_archive.py ../OrthodoxPrayers-5.0.16-R20-FULL-ROOT.zip --root-layout
```

لإنشاء تحديث R19.1 يُفك مباشرة في جذر مستودع موجود، دون مجلد غلاف:

```bash
python scripts/create_r19_root_patch.py ../OrthodoxPrayers-5.0.15-R19.1-root-patch.zip
```

## حدود النسخة الحالية

- لقطة المصدر المضمّنة تبقى نسخة كاملة موقعة تصلح للرجوع دون إنترنت، وليست وعدًا بأنها بيانات تاريخ تشغيلك.
- مهمتا Debug وRelease تستوردان أحدث بيانات منشورة موقعة من فرع `verified-data` بعد نجاح بوابة المصدر، وتتحققان منها قبل بناء التطبيق.
- لا تُعد أي لغة مكتملة كنسيًا بعد. لا يجوز تغيير الحالة إلى مكتملة إلا بعد استيراد طبعة أصلية كاملة ومصرح بها لكل خدمة ولغة.
- نص القداس الإنجليزي واليوناني المستورد من طبعة 2017 يتطلب الاحتفاظ بإثبات الإذن الخطي خارج المستودع العام.
- حزمة الكتاب المقدس المضمّنة تغطي قراءات المرشح الحالي؛ ويستطيع خط التحديث جلب Corpus أوسع من المصادر المسجلة عند توفر الشبكة.
- لا ترفع أي ملف مفتاح خاص أو Keystore أو كلمات مرور إلى GitHub.
