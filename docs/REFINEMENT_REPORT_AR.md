# تقرير تنقيح نسخة المصدر

## تدقيق R20

- أُلغي اعتبار امتلاء الحقول دليلًا على اكتمال الخدمة الكنسية.
- أضيفت قائمة 15 خدمة إلزامية وحاجز Production يفشل عند أي نقص.
- الحالة الحالية الصادقة هي 0/15 خدمة مثبتة كاملة في كل لغة، مع بقاء الحقول التقنية المعبأة قابلة للفحص.
- صارت ملفات اليوم الموقعة معزولة في محتواها ودليل مصدرها، لا في الرابط والتوقيع فقط.
- أضيفت مواضع دلالية لقراءات القداس في الطبعات العربية والإنجليزية واليونانية.
- أصلح تنظيف `\+w` في USFM، وأضيف تنظيف عرض دفاعي للقطات الموقعة القديمة.
- أزيلت تسمية «القداس الكامل»، وصُحح حرف لاتيني متسرب في نص يوناني.

## ما تم إصلاحه

- إزالة سجل Git والكاش وملفات Python bytecode وملفات البناء من نسخة التسليم.
- حذف رسائل Commit وسكربتات التحقق الخاصة بالإصدارات القديمة.
- نقل تقارير الإصلاح وملاحظات الإصدارات التاريخية إلى `docs/history`.
- تفعيل `--strict-native-lanes` ليشغّل فحصًا حقيقيًا يمنع نسخ العربية إلى الإنجليزية أو اليونانية ويرفض الأبجدية الخاطئة.
- إبقاء اكتمال الإنجليزية واليونانية شرطًا منفصلًا لإصدار Production، دون اختلاق نصوص أو ترجمتها.
- استبدال Gradle Wrapper JAR وسكربتي Linux/Windows بالملفات الرسمية المتوافقة مع Gradle 8.13.
- إضافة `scripts/verify_gradle_wrapper.py` لفحص بصمات Wrapper والتوزيعة قبل البناء.
- نقل سر توقيع البيانات إلى GitHub Environment باسم `production-data-signing` في Workflow التحديث.
- فصل توليد البيانات وفحصها عن التوقيع: لا يُستعاد المفتاح الخاص إلا بعد انتهاء جميع عمليات جلب المصادر والتحقق غير الموقّع، ويُحذف قبل Commit أو Push.
- حماية مستورد USFM من Zip Bomb عبر حدود لعدد الملفات والحجم غير المضغوط الإجمالي وحجم الملف ونسبة الضغط والمسارات المشبوهة.
- تحسين مولّد ZIP ليكون حتميًا ويمنع مسارات الهروب والتسريبات والملفات السرية والكاش، ويحافظ على صلاحية تشغيل `gradlew`.
- جعل بناء Debug يستورد أحدث بيانات منشورة موقعة بعد نجاح بوابة المصدر ويتحقق منها قبل إنشاء APK.
- تحديث README ووثائق الجاهزية والحماية.

## الحالة التاريخية بعد R19

- لقطة المصدر المضمّنة الموقعة تحمل تاريخ 2026-07-17 وتبقى مرجع الرجوع دون إنترنت؛ يستورد بناء Debug وRelease أحدث لقطة منشورة وموقعة قبل التجميع.
- كانت حزم الخدمات تُوصف بأنها مكتملة بحسب عدّ الحقول: العربية 533/533، الإنجليزية 770/770، واليونانية 762/762. أثبت تدقيق R20 أن هذا قياس تقني وليس اكتمالًا كنسيًا.
- تتحقق بوابة Production بصورة مستقلة من نص الرسالة والإنجيل ومصدرهما وتوقيعهما، ولا تعتمد على نسبة الواجهة وحدها.
- لم تُضف نصوص دينية غير موثقة، ولم تُستخدم ترجمة آلية لسد النقص.
- لم يُبنَ APK محليًا في بيئة التنقيح لعدم توفر توزيعة Gradle واعتماديات Android في الكاش؛ Workflow GitHub يبني Debug ويفحص Lint بعد الرفع.

## إصلاح توافق GitHub Actions لنهايات الأسطر

تم تعديل فحص بصمات سكربتات Gradle بحيث يحسب البصمة بعد توحيد نهايات الأسطر إلى LF.
هذا يمنع الاختلاف الكاذب بين نسخة ZIP ونسخة GitHub checkout، حيث تفرض `.gitattributes`
صيغة CRLF على `gradlew.bat` بينما قد يبقى الملف LF خارج Git. لا تتغير بصمة
`gradle-wrapper.jar` أو بصمة توزيعة Gradle؛ التطبيع يطبق على السكربتات النصية فقط.
