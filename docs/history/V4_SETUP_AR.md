# إعداد Orthodox Prayers 4.0.0

هذه النسخة تستخدم زوج توقيع واحد فقط. المفتاح العام موجود داخل المشروع والتطبيق، والمفتاح الخاص المرفق خارج ZIP يجب وضعه فقط في GitHub Actions Secret.

## الرفع بواسطة GitHub Desktop

1. فك ضغط `orthodox_prayers-v4.0.0.zip`.
2. افتح GitHub Desktop ثم `Repository → Show in Explorer`.
3. انسخ جميع محتويات مجلد `orthodox_prayers-v4.0.0` إلى مجلد المستودع واختر Replace.
4. تأكد أن `.github/workflows` يحتوي فقط `build.yml` و`update.yml`.
5. اعمل Commit ثم Push باستخدام الرسالة المرفقة.

## استبدال Secret القديم

احذف Secret القديم `DATA_SIGNING_PRIVATE_KEY_B64` ثم أنشئه من جديد بنفس الاسم، والصق كامل السطر الموجود في الملف الخارجي:

`OrthodoxPrayers-v4.0.0-DATA_SIGNING_PRIVATE_KEY_B64.txt`

لا ترفع ملف المفتاح أو ملف Base64 إلى المستودع.

Workflow التحديث يشتق المفتاح العام من الـSecret ويقارنه بالمفتاح العام داخل التطبيق قبل إنشاء أي بيانات. إذا كانا غير متطابقين يتوقف برسالة واضحة قبل التوقيع، لذلك لا يمكن أن يتكرر خطأ `bad signature` بصمت.

## التشغيل الأول

بعد Push:

1. شغّل `Build` وانتظر نجاح Debug APK.
2. شغّل `Update` يدويًا مع `mode=update` والتاريخ فارغًا.
3. بعد نجاحه، شغّل `Update` مع `mode=verify` أو انتظر فحص 00:15.
4. أغلق التطبيق وافتحه أو اضغط تحديث البيانات.

## الجدول

- 00:00 بتوقيت عمّان: إنشاء بيانات اليوم وتوقيعها ونشرها.
- 00:15 بتوقيت عمّان: التحقق من التاريخ والتوقيع.

إذا نقص نص ديني رسمي، يُنشر باقي اليوم الموثوق ويظل ذلك النص وحده فارغًا، دون ترجمة أو تخمين.
