# تحصين Orthodox Prayers 4.1.5-r3

## ما تغيّر

1. **عزل مفتاح توقيع البيانات:**
   - توليد بيانات اليوم ومسارات اللغات يتم بوضع `--unsigned` دون وجود المفتاح الخاص على القرص.
   - أي توقيع قديم بجوار JSON جديد يُحذف ويُرفض أثناء التحقق غير الموقّع.
   - بعد نجاح جلب المصادر والتحقق البنيوي، يُستعاد المفتاح داخل GitHub Environment المحمي.
   - تُوقّع البايتات النهائية داخل شجرة النشر نفسها، ثم يُحذف المفتاح قبل Commit أو Push.

2. **حماية أرشيفات الكتاب المقدس:**
   - حد أقصى لحجم التنزيل المضغوط.
   - حد أقصى لعدد الملفات داخل ZIP.
   - حد لحجم كل ملف بعد فك الضغط وللمجموع غير المضغوط.
   - رفض نسبة الضغط الخطرة، والملفات المشفرة، ومسارات `..` أو المسارات المطلقة.

3. **توافق صلاحية Gradle Wrapper مع GitHub وWindows:**
   - تُطبع صلاحية `gradlew` تلقائيًا إلى `0755` قبل بدء اختبارات Python.
   - يعمل الإصلاح حتى عندما يرفع GitHub Web أو Windows الملف بوضع `100644`.
   - يفرض مولّد ZIP الصلاحية التنفيذية داخل الحزمة بصورة مستقلة عن صلاحية نسخة العمل.

4. **حزمة مصدر نظيفة:**
   - استبعاد `.git` والكاش وملفات bytecode والبناء والمفاتيح وملفات البيئة ورسائل Commit.
   - فحص المسارات والروابط الرمزية والأسماء الحساسة بعد إنشاء ZIP.
   - حفظ `gradlew` بصلاحية تشغيل Unix وإنشاء ملف SHA-256 جانبي.

5. **بيانات Debug الحديثة:**
   - تبقى لقطة كاملة موقعة داخل المصدر لتشغيل التطبيق دون إنترنت.
   - بعد نجاح بوابة المصدر، يستورد Workflow البناء أحدث بيانات موقعة من `verified-data` ويتحقق منها قبل إنشاء Debug APK.

## الاختبارات

أضيفت اختبارات رجعية تمنع عودة المشكلات السابقة، وتشمل Zip Bomb، ومسارات ZIP الخطرة، والتوقيعات القديمة، وحتمية ZIP النظيف، وصلاحية `gradlew`. المجموعة الحالية: **84 اختبار Python**.

## ما لم يتغير

بوابة Production ما زالت محجوبة عمدًا حتى اكتمال محتوى الخدمات الإنجليزية واليونانية والمصادر الأصلية المطلوبة. لم تُضف ترجمة آلية أو نصوص دينية غير موثقة.
