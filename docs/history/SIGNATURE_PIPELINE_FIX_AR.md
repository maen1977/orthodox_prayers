# إصلاح خط نشر وتوقيع بيانات اليوم

## المشكلة
كان ملف `today.json` يُوقّع في نسخة العمل، ثم تُنسخ الملفات إلى فرع `verified-data` وتُضاف إلى Git. يمكن لمعالجة Git للملفات النصية أو بقايا توقيع قديم في الفرع أن تجعل البايتات المنشورة مختلفة عن البايتات التي تم توقيعها، فتظهر رسالة `bad signature` بعد الاستنساخ.

## الإصلاح
- اعتبار ملفات JSON الموقعة وملفات `.sig` والمفتاح العام ملفات ثنائية في `.gitattributes` لمنع أي تحويل لنهايات الأسطر.
- تنظيف محتوى حزمة `verified-data` قبل كل نشر حتى لا يبقى توقيع قديم.
- حذف جميع التواقيع المنسوخة وإعادة توقيع الملفات داخل مجلد النشر نفسه.
- التحقق من التوقيعات قبل الـCommit.
- إنشاء أرشيف من الـCommit نفسه والتحقق من البايتات المخزنة في Git قبل الـPush.
- إبقاء فحص إضافي بعد الاستنساخ من GitHub.

لا يلزم تغيير `DATA_SIGNING_PRIVATE_KEY_B64` ما دام فحص تطابق المفتاح ينجح.
