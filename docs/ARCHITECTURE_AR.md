# بنية التطبيق 4.2.0

## مبدأ المحتوى

العربية والإنجليزية واليونانية ثلاث قنوات مستقلة. لا يترجم البرنامج النصوص الدينية بين اللغات، ولا يستخدم الذكاء الاصطناعي لإعادة الصياغة أو التشكيل، ولا يعرض لغة بديلة عند غياب النص المطلوب. النص المعروض محفوظ حرفيًا، بينما يُنشأ فهرس بحث مشتق ومنفصل لا يغيّر النص الأصلي.

## سلسلة البيانات اليومية

1. `update_liturgical_data.py` يولّد هيكل اليوم بتوقيت عمّان والتقويم المعتمد.
2. `orthodox_integrity.py` يتحقق من مراجع اليوم وترتيب المصادر الرسمية.
3. `fill_daily_from_native_corpora.py` يحل المرجع من مكتبة كتاب مقدس رسمية مستقلة للغة نفسها، وبقاعدة الكل أو لا شيء.
4. `enforce_native_daily_lanes.py` يزيل أي نص قديم أو مخالف لمسار اللغة ويثبت سياسة عدم الترجمة وعدم الـfallback بين اللغات.
5. تُفحص البنية والمصادر والبصمات والتوقيع، ثم يُنشر إلى `verified-data` فقط عند نجاح البوابة كاملة.
6. التطبيق يتحقق من التوقيع ويحتفظ بآخر نسخة سليمة إذا فشل التحديث.

## طبقات النص

- `data/scripture/native/<lang>`: مكتبات الكتاب المقدس الأصلية المعتمدة لكل لغة. حالتها الحالية فارغة إلى أن تُستورد نسخة رسمية مأذون بها.
- `data/services/native`: حزم الصلوات والخدمات المستقلة لكل لغة.
- `app/src/main/assets/data/search`: فهارس بحث مشتقة تحفظ النص الأصلي وبصمته.
- `canonical/source_native_contract.json`: العقد الملزم للمصادر واللغات والمحافظة الحرفية.

## التطبيق

- `OrthodoxPrayersApp`: يملك Repository واحدًا.
- `DataRepository`: تحميل البيانات والتحقق واختيار حزمة اللغة نفسها فقط.
- `DailyDataStore`: كتابة ذرية مع نسخة احتياطية وآخر نسخة سليمة.
- `VerifiedContentSanitizer`: طبقة دفاع إضافية ترفض النصوص الكتابية غير المثبتة.
- `UpdateCoordinator`: فحص منتصف الليل والتأكيد عند 00:15 بتوقيت عمّان.
- `ReaderScreen` و`ReaderAdapter`: عرض طويل دون صفحة بيضاء أو خلط لغات.

## Workflows

يوجد ملفان فقط:

- `build.yml`: بوابة الجودة وUnit Tests وLint Debug وبناء Debug APK في خطوات منفصلة؛ ولا يسمح بإصدار موقّع إلا بعد اكتمال المحتوى الأصلي المطلوب ونجاح `validate_release_readiness.py`. CodeQL والمحاكي متوقفان مؤقتًا حتى يستقر البناء الأساسي.
- `update.yml`: توليد يومي عند 00:00 بتوقيت عمّان، توقيع البيانات وبيان التحديث، والتحقق من النسخة المنشورة إلى `verified-data` ضمن التشغيل نفسه.

## النسخ المتطابقة المقصودة

توجد نسخ `today.json` وحزم البيانات داخل Android assets للعمل دون إنترنت. الاختبارات تمنع اختلاف النسخ الضرورية، بينما تُستبعد المفاتيح الخاصة والكاش والبناء والتقارير من المستودع والأرشيف.

## جدولة التطبيق

يستخدم تطبيق Android مهمة WorkManager واحدة عند 00:05 بتوقيت عمّان مع شرط الشبكة وإعادة المحاولة. لا يحتاج تحديث المحتوى إلى إذن المنبهات الدقيقة. ويتحقق التطبيق من بيان تحديث موقّع قبل تنزيل مسار اللغة المختار.
