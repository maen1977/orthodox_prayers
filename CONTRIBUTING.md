# المساهمة

1. افتح Pull Request لتغييرات الكود، ويفضل حماية `main` من الدفع المباشر.
2. شغّل `python scripts/run_quality_gate.py --strict-native-lanes` واختبارات Gradle وLint.
3. لا تعدّل نصًا دينيًا مقفلًا أو بصمته أو توقيعه يدويًا.
4. كل نص جديد يحتاج مصدرًا رسميًا مسجلًا من اللغة نفسها ورابطًا وبصمة وإذنًا.
5. لا تُستخدم الترجمة بين المسارات، ولا الذكاء الاصطناعي، ولا التشكيل الآلي للنصوص الدينية.
6. لا تنشر مقطعًا كتابيًا جزئيًا، ولا تعلن نصًا مختصرًا كاملًا.
7. عند غياب المصدر أو تعارضه يفشل التحديث مغلقًا وتبقى آخر نسخة موقعة سليمة.
8. لا تضع مفاتيح خاصة أو Keystores أو كلمات مرور أو ملفات موافقات في Git أو سجلات Actions.
9. حافظ على Android API 26 كحد أدنى، ودعم RTL وإمكانية الوصول.
10. قبل Tag إنتاجي شغّل `python scripts/validate_release_readiness.py`.
