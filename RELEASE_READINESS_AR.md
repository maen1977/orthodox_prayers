# حالة جاهزية Orthodox Prayers 4.1.5 — المصدر المنقح

## جاهز الآن

- المشروع جاهز للرفع إلى GitHub.
- لا يحتوي المشروع العام على مفتاح بيانات خاص أو Android Keystore.
- يوجد Workflowان فقط: `Build` و`Update`.
- التحديث مضبوط على 00:00، والتحقق على 00:15 بتوقيت عمّان.
- التوقيع والتحقق وLast-known-good مفعلة.
- اللغات الثلاث مفعلة دون fallback بين اللغات.
- البحث يستخدم فهرسًا مشتقًا ويعرض النص الأصلي دون تغيير.
- اختبارات Python: 84/84 ناجحة في النسخة المنقحة.

## غير جاهز بعد لإصدار Production مكتمل

- حزمة الخدمات الإنجليزية: 132/601.
- حزمة الخدمات اليونانية: 132/601.
- مكتبات الكتاب المقدس الرسمية الكاملة للعربية والإنجليزية واليونانية غير مستوردة بعد.
- لذلك لا تتوفر نصوص رسالة وإنجيل اليوم الأصلية في المسارات الثلاثة داخل النسخة الحالية.

هذا النقص لا يؤدي إلى ترجمة أو خلط؛ يظهر النص غير متوفر.

## بوابة الإنتاج

Workflow `Build` يسمح ببناء Debug للاختبار بعد فحص سلامة المصدر، ثم يستورد أحدث بيانات موقعة من `verified-data` ويتحقق منها قبل التجميع. أما مهمة الإصدار الموقّع فتنفذ:

```bash
python scripts/validate_release_readiness.py
```

وستفشل عمدًا إلى أن تكتمل المتطلبات السابقة. بعد استيراد المحتوى الصحيح يعاد تشغيل البوابة دون تغيير النصوص يدويًا داخل Workflow.

## تحقق البناء

لم يُنفذ بناء Android الكامل محليًا إذا لم تتوفر حزمة Gradle/Android SDK في البيئة. Workflow `Build` ينفذ حاليًا Unit Tests وLint Debug وبناء Debug APK في خطوات منفصلة بعد الرفع. يجب إعادة اختبار المحاكي وCodeQL قبل إعلان Production عندما يصبح البناء الأساسي مستقرًا.


## حماية مفاتيح النشر

- مفتاح توقيع بيانات اليوم يوضع داخل GitHub Environment باسم `production-data-signing`، ولا يُكتب إلى القرص إلا بعد انتهاء تنزيل المصادر وفحص المخرجات غير الموقعة.
- مفتاح توقيع Android يوضع داخل Environment منفصل باسم `production`.
- يجب ضبط قواعد الحماية والمراجعين والفروع المسموحة من إعدادات GitHub، ولا تكفي كتابة اسم Environment داخل YAML وحدها.
