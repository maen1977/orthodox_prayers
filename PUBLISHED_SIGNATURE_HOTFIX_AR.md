# إصلاح توقيعات ملفات اللغات عند النشر

كان التحديث يوقّع ملفات اللغات في شجرة المصدر، ثم ينسخها إلى فرع `verified-data`. ظهر في النشر الفعلي أن ملفات JSON لليونانية والإنجليزية تغيّرت بينما بقيت ملفات التوقيع من الإصدار السابق، لذلك فشل التحقق بعد إعادة استنساخ الفرع.

الإصلاح:

- إعادة توقيع كل ملفات اللغات الموجودة داخل شجرة النشر نفسها قبل إنشاء الـcommit.
- التحقق من التوقيعات مباشرة داخل شجرة النشر.
- إنشاء أرشيف من `HEAD` والتحقق من بايتات Git الفعلية قبل الدفع إلى GitHub.
- تعطيل أي تطبيع نصي لملفات JSON الموقعة وملفات `.sig` عبر `.gitattributes`.
- إبقاء التحقق من الفرع البعيد بعد الدفع كطبقة حماية أخيرة.

بعد رفع التعديل شغّل Workflow باسم **Update** مع `mode: update`.
